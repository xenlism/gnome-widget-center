# CONTRIBUTING.md — วิธีทำงานแบบแบ่งเป็น task ให้ AI หลายตัว/หลายวัน

## แนวคิด

แทนที่จะยัดทั้งโปรเจกต์เข้า context เดียว ให้แบ่งเป็นไฟล์ `development/tasks/NN-ชื่องาน.md`
ที่แต่ละไฟล์ **สมบูรณ์ในตัวเอง** (self-contained) — เปิด session ใหม่ใน AI ตัวไหนก็ได้
วางแค่ prompt สั้น ๆ + ไฟล์ที่เกี่ยวข้อง ก็ทำงานต่อได้ทันที ไม่ต้องรู้ history ทั้งหมด

## วิธี kick-off งานแต่ละ task ให้ AI (เทมเพลต prompt)

```
อ่าน development/docs/ARCHITECTURE.md และ development/docs/WIDGET_API.md (ถ้าเกี่ยวกับ widget)
แล้วทำงานตาม development/tasks/03-settings-store.md เท่านั้น
ไฟล์ที่แก้ได้: ตามที่ระบุใน "Files to touch" ของ task นี้เท่านั้น
ห้ามแก้ไฟล์อื่นนอกเหนือจากนี้ ถ้าจำเป็นต้องแก้ให้หยุดถามก่อน
เสร็จแล้วสรุปว่าทำอะไรไปบ้าง + acceptance criteria ข้อไหนผ่าน/ไม่ผ่าน
```

## กติกาการแบ่งงาน

1. **ทำตามลำดับเลขใน `development/tasks/`** — งานเลขหลังมัก depend on เลขก่อนหน้า (ระบุไว้ใน
   "Depends on" ของแต่ละ task) แต่บาง task ทำขนานกันได้ (ระบุไว้เช่นกัน)
2. แต่ละ task ควรจบได้ภายใน 1 session/1 วัน (ถ้าใหญ่เกินไปให้ย่อยเพิ่มเป็น
   `03a-...md`, `03b-...md`)
3. เสร็จ task ไหน ติ๊ก checkbox ใน `development/tasks/ROADMAP.md` และเขียนบันทึกสั้น ๆ ว่า
   ตัดสินใจอะไรเพิ่มเติม (ถ้ามี) ไว้ท้ายไฟล์ task นั้น ใต้หัว `## Notes from implementation`
4. ถ้า AI ตัวหนึ่งค้นพบว่าต้องแก้ไฟล์นอกเหนือจากที่ระบุ → ให้หยุดและรายงาน
   ไม่ใช่แก้เองเงียบ ๆ (กัน AI สองตัวชนไฟล์กันในคนละ session)
5. ทุก task ที่แตะ `products/extension/` (core) ต้องรักษา "no-op สำหรับ widget ที่มีอยู่" — เทส
   ด้วย widget ตัวอย่าง `widgets/clock/` เสมอหลังแก้ core

## โครงสร้างของแต่ละไฟล์ task

- **Goal** — เป้าหมาย 1-2 บรรทัด
- **Depends on** — task ไหนต้องเสร็จก่อน
- **Context** — สรุปสิ่งที่ต้องรู้ ไม่ต้องเปิดไฟล์อื่นอ่านเพิ่ม
- **Files to touch** — ไฟล์ที่อนุญาตให้สร้าง/แก้
- **Steps** — ขั้นตอนแนะนำ (ไม่บังคับตายตัว)
- **Acceptance criteria** — เช็คลิสต์ว่าทำเสร็จจริงเมื่อไหร่
- **Out of scope** — กันบาน (สิ่งที่ห้ามทำใน task นี้ ผลักไป task อื่น)

## เมื่อสลับ "AI ตัวใหม่" กลางทาง

ให้แนบสรุปสถานะปัจจุบันจาก `development/tasks/ROADMAP.md` (ส่วน checklist) เสมอ เพราะไฟล์นี้คือ
"source of truth" ว่าถึงไหนแล้ว — ไม่ต้องพึ่งความจำของ AI ตัวเดิม
